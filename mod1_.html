MY INITIAL POST

    AUTHENTICATION AND AUTHORIZATION; AN ESSENTIAL FOR NETWORK AND SOFTWARE SECURITY.

    Security is a crucial part of any application or environment (Bullock and Benford, 1999) and granting access to these secured applications and the environment needs to be controlled; who gets access to what? What are the rules? How is it tracked? These are managed through an access control system providing access to restricted areas of the business (SailPoint, 2021).” Access control is governed according to the space within which subjects and objects reside, and the ability to traverse space to get close to an object” by Bullock and Benford, 1999. Who(subject) gets access to resources(object) is validated through authentication and authorisation processes.

    Authentication, verifying who someone is, has long passed username and passwords to more advanced Artificial intelligence and deep learning technologies. The application of multi-factor authentication verifying an individual identity based on two to three different parameters, something you (have, know and are) works based on artificial intelligence and deep learning, where data, logs, transactions, and real-time communications are analysed to detect an intruder (Okta, 2018).

    Hardware embedded chips are evolving technologies that authenticate users using multiple levels and methods. An example is the sixth generation vPro Chips by intel (IFF Lab, 2019).

    Authorisation is to which asset one has permission to access through settings supported by security teams (Okta, 2018). The more often used to perform these authorisation processes include Role-Based Access Control, Discretionary based access control, Attribute-based access control, Rule-Based Access Control. These strategies help application developers control the access to the resources depending on the applied method and to what level the user is permitted.

    Authentication and authorisation are vital in access control management, verifying a user’s identity effectively, thus allowing authenticated users to access protected resources depending on the given authority level; these ensure that security is maintained both in the network and the software to be accessed (Wang, Bo and Zhang, 2013).

Hi, I agree with your post. Firstly, the importance of authentication and authorisation is well documented. Summary Post

The importance of control mechanisms (how, who, and to what extent) access is granted into a network cannot be overemphasised, authenticating a user into the network, as discussed during this period. One of the weakest links in security is the user; as pointed out by Richard (Tolofari and Meadows, 2021), users need to be educated on how to create their passwords, not using conventional passwords like 12345. The Open Web Application Security Project (OWASP), among other bodies, provides guidance on minimum criteria for a strong password, that passwords cannot be shorter than eight and should be enforced by applications (NIST SP800-63B). Maximum password length should not be set too low and more; ensure credentials change when there is compromise or password leak. A password alone cannot guarantee the security of a network; the combination of other factors- multifactor authentication mechanisms should be implemented." The potential in multifactor authentication security features will not be exhausted anytime soon as long as there are unique and differing attributes in life. The challenge may only lie in the ability to integrate new identification features into authorisation and authentication" by Jubet., Authentication methods are based on only the possession of credentials issued to the user. The verification is performed on credentials offered at present, not considering how or when the user obtained the credentials (Hasan and Khan, 2015). Nitin also pointed out during the discussions the importance of Adaptive Multifactor Authentication. Where contexts like location awareness, device fingerprinting or impossible velocity are added to the multifactor authentication (Stjernlof and Vries, 2020), these adaptive multifactor authentications using artificial intelligence is a step further to harden the authentication system. Isioma, during the discussions, highlighted that authenticating a user before authorisation; granting access to data or resources in the network is crucial, not the other way round. To what criteria and rules are the users assessed (Tolofari and Okafor -Tolofari, 2020), like Discretionary access control: where the owner of the system determines who gets access to the system; Role-based access control, where access is granted based on the user role, Mandatory access control, which allows access based on clearance level are all among other criteria to authorise a user on what resources they can access in the network. Authentication and authorisation are essential to network security, how they are implemented, and to what extent should depend on the asset's value. The efficiency depends on the technology used to enforce this authentication and authorisation process.
